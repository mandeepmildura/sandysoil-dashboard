How will you set program start times?

Two options — pick the one that fits how you actually work.

A

Time picker in program settings (recommended to start)

When you create or edit a program (e.g., "Avocados"), you type the start time and pick the days. The Calendar page then shows all programs as colour-coded blocks on a timeline so you can see overlaps at a glance — but scheduling is done via inputs, not dragging.

What you'd see:

AVOCADOS PROGRAM

Start time 04:00
M W F T S

CITRUS PROGRAM

Start time 06:00
M W F T S

✓ Build in one session · no extra libraries · works on mobile

B

Drag-to-place on a day timeline

The Calendar page shows an hourly day view. You drag a program block left/right to set its start time. Duration is shown by block width — zone cards inside show per-zone time allocation. Like Google Calendar, but for irrigation programs.

⚠ Needs extra drag library or custom pointer events · complex on touch/mobile · 2–3× build time

My suggestion: Ship A now — time picker with a visual day-timeline on the Calendar page so you can see all your programs laid out. Dragging (B) can be added later once the scheduling model is solid. The hard part is the DB + scheduler logic, not the time input widget.

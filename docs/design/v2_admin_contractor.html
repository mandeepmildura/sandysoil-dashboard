Why this view

When you have 5+ customers, "tabs across farms / devices / customers" stops scaling. The contractor view collapses everything into one scannable table where you can see every customer's health at a glance — pressure, last-seen, scheduled-vs-actual runs, firmware version. Hunter Hydrawise calls this the Contractor Dashboard; Stripe calls it Connect; Shopify calls it Partner Dashboard. Same idea: one row per customer, sort/filter, click to drill into their farm.

The status pill (Healthy / Warning / Offline) is the most important column — it's how you triage which customer to call back first. Same colour system as the customer-facing dashboard so terminology stays consistent across personas.
